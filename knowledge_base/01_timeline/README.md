## How To Handle Relevance

Do not delete older material. Instead, classify it.

- `historical foundation`: important for original scope and commitments
- `transition snapshot`: useful for understanding what changed
- `current planning`: the documents that should drive the next implementation steps

## Authority Rule

When documents disagree:

1. newer phase beats older phase
2. current code beats docs for already-implemented offline behavior
3. hardware-confirmation notes beat older assumptions
